# Kayan Contracting Ltd. -- Pillar-Cluster Content Architecture

**Prepared for:** Eric (AI Precision Marketing) & Bryan Dewey (Kayan Contracting)
**Date:** March 29, 2026
**Subject:** Complete pillar-cluster blog strategy for kayancontracting.ca

---

## Executive Summary

This document defines a 4-pillar, 38-post content architecture designed to establish Kayan Contracting as the topical authority for roofing in Stony Plain, Spruce Grove, and Parkland County. Each pillar page (3,000-5,000 words) acts as a comprehensive hub, with 6-12 cluster posts (1,200-2,500 words each) targeting specific long-tail keywords and linking back to the pillar.

**Why this works in 2026:** Google's Helpful Content system evaluates the depth and breadth of topic coverage. A site with 20+ interconnected articles on roofing consistently outranks a site with one long guide. Intent-based clustering (not keyword stuffing) is what builds topical authority. Each cluster post must be more comprehensive on its subtopic than the pillar page section covering the same ground -- otherwise it creates cannibalization rather than authority.

**Current state:** The site has 3 existing blog posts (roofing materials guide, metal vs shingles, thermal imaging) plus 3 location pages (Stony Plain, Spruce Grove, Parkland County). This strategy builds on that foundation.

## URL Structure

All blog content follows a flat `/blog/` structure with SEO-friendly slugs:

```
/blog/                                          -- Blog index
/blog/complete-guide-roofing-stony-plain        -- Pillar 1
/blog/metal-roofing-alberta-guide               -- Pillar 2
/blog/hail-damage-roof-insurance-alberta        -- Pillar 3
/blog/[topic-slug]                              -- Cluster posts

/services/stony-plain                           -- Location pillar (exists)
/services/spruce-grove                          -- Location pillar (exists)
/services/parkland-county                       -- Location pillar (exists)
/services/edmonton                              -- Location pillar (exists)
/services/st-albert                             -- Location pillar (NEW)
/services/lac-ste-anne                           -- Location pillar (NEW)
```

**URL rules:**
- Lowercase, hyphens only, no trailing slashes
- Include primary location when relevant (not forced)
- Keep under 60 characters when possible
- Never change a URL after publishing -- set redirects if needed

```
    ┌───────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐
    │   PILLAR 1    │  │  PILLAR 2    │  │  PILLAR 3    │  │   PILLAR 4       │
    │   Roofing     │  │  Metal       │  │  Hail/Ins.   │  │   Location       │
    │   Guide       │◄─┤  Roofing     │◄─┤  Claims      │◄─┤   Pages          │
    │   (Hub)       │──►│  (Hub)      │──►│  (Hub)      │──►│   (Hub)          │
    └───┬───┬───┬───┘  └──┬───┬──┬───┘  └──┬──┬───┬───┘  └──┬──┬───┬───┬───┘
        │   │   │         │   │  │         │  │   │         │  │   │   │   │
       10 cluster        8 cluster        7 cluster       6 location pages
       posts             posts            posts           + cross-links
```

**Linking rules:**
1. Every cluster post links back to its parent pillar (required, in first 200 words)
2. Every cluster post links to 2-3 other cluster posts within the same pillar
3. Every cluster post links to 1-2 cluster posts in OTHER pillars when topically relevant
4. Pillar pages link to every cluster post in their cluster
5. Pillar pages link to each other where content overlaps
6. Location pages link to relevant cluster posts and pillar pages
7. All posts link to at least one service page (commercial intent bridge)
8. Anchor text uses the target keyword of the destination page (not "click here")

## Schema Markup Strategy

Every blog post and pillar page should include:

1. **Article schema** (BlogPosting) with headline, datePublished, author, image
2. **FAQ schema** (FAQPage) on posts with FAQ sections -- this drives featured snippets
3. **BreadcrumbList schema** showing Home > Blog > [Post Title]
4. **LocalBusiness schema** on location pages (already exists on some pages)

---

## Content Quality Standards

Each post must meet these criteria before publishing:

1. **Unique depth** -- Covers its subtopic more comprehensively than the pillar page section on the same subject
2. **Alberta-specific** -- Not generic roofing content. References local climate, hail belt, chinooks, freeze-thaw, specific cities
3. **Real photos** -- Kayan project photos, not stock images. Thermal images where relevant
4. **First-person expertise** -- Written from Kayan's perspective ("In our 22 years...")
5. **FAQ section** -- Every post ends with 3-5 FAQs using natural language (voice search optimization)
6. **CTA** -- Every post has a clear, contextual call-to-action (not generic)
7. **Internal links** -- Minimum 3 internal links per post (1 to parent pillar, 2 to related clusters)
8. **Meta description** -- 155 characters max, includes target keyword, compelling to click
9. **Alt text** -- Every image has descriptive, keyword-inclusive alt text
